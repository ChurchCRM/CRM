# 🌸 ChurchCRM 7.4.2 — The "Path Forward & Protected Access" Release

**Release Date**: July 12, 2026  
**Theme**: Smooth Migrations, Hidden Menus, and Airtight Permissions

**Version 7.4.2** is a landmark release dedicated to two critical areas: ensuring legacy **ChurchInfo** databases have a flawless upgrade path to modern ChurchCRM, and heavily refining our daily user permissions so your team only sees exactly what they are authorized to access.

---

## 🗺️ A Clear Path Forward for Legacy & ChurchInfo Congregations
If your church is still running on older software or ancient versions of ChurchCRM, we have built a highway to bring you into the modern era safely.

* **Flawless ChurchInfo Imports:** We have resolved a long-standing roadblock for churches migrating from **ChurchInfo 1.3.1**. When restoring these historic backups, the system now automatically detects and upgrades old MD5-encrypted passwords, ensuring your users can log in to the new system immediately without manual resets.
* **Consolidated Upgrade Steps:** We have condensed 29 individual, historic migration steps into a single, highly efficient process. This significantly reduces the risk of database timeout errors when jumping from legacy versions to 7.4.2.
* **Redesigned Upgrade Wizard:** Before you click upgrade, you will now see an interactive "Upgrade Path Panel," a target version selector, and the official release notes directly inside the application so you know exactly what is changing.

---

## 🛡️ Cleaner UI, Smarter Permissions
We’ve removed the visual clutter and improved security by ensuring users only see the buttons and pages their roles allow.

* **"If You Can't Access It, You Won't See It":** The main sidebar and menus are now highly permission-aware. If a staff member or volunteer doesn't have authorization to view a specific tool (like finances or advanced settings), that menu entry is completely hidden from their screen.
* **Zero-Permission Safety Net:** For users with no special permissions assigned, we have established a safe "baseline" read-only access to basic household and profile rosters. This keeps them connected to the church directory without giving them any modification powers.
* **Strict Finance Guards:** We have added heavy permission blockades to all pledge and fundraising pages. Only designated financial officers can access or edit these sensitive pages.

---

## 👤 Simplified Password & User Auditing
Administrators now have a much clearer birds-eye view of who has access to the system and whose accounts need attention.

* **User Status & Password Audits:** A brand-new "Status" column has been added to the Admin Users screen. At a glance, you can see if an account is active, locked, or if they have a pending "Password Change Required" notice.
* **Clean Reset Screens:** We fixed a visual glitch on the forced password reset page. If a user is required to change their credentials, they are now presented with a clean, beautifully formatted card layout that makes the process effortless.

---

## 🌍 Continual Global Polish
* **Always Localized:** Even as we adjust security menus and upgrade pathways, our global community has translated all 42 supported languages to keep these new features easily readable for your local leadership.

---

## ⚠️ Important Reminder
* **Server Requirement:** This version continues to **require PHP 8.4**.
* **Guided Upgrade:** Our newly polished upgrade wizard is standing by to guide you securely from 7.4.1 to 7.4.2.
